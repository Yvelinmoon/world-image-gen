# 已有 IP 参考图流程

## 为什么需要参考图

不要假设模型熟悉任何 IP 的视觉形象——即使是知名 IP，模型内化的表现也可能不准确，依赖 IP 名称往往生成泛化结果而非原版质感。**参考图是最可靠的还原手段。**

## Step 1：搜索参考图

```
WebSearch: "[IP名称] [角色名] official art / concept art"
WebSearch: "[IP名称] [角色名] 官方设定图 / 游戏截图"
```

优先选：官方宣传图、概念设定图、高清游戏截图、模型渲染图。
避免：同人图（风格不一致）、低分辨率缩略图。

## Step 2：下载并作为参考图传入

```bash
# nano-banana
curl -fsSL "[图片URL]" -o /tmp/ip-ref.jpg
nano-banana "[prompt]" -r /tmp/ip-ref.jpg -a [比例] -s 2K -m pro -o [输出名]
rm /tmp/ip-ref.jpg
```

## Step 3：prompt 里描述视觉，不只写名字

即使有参考图，也要在 prompt 中明确写出角色的视觉特征，不能只写角色名：

```
[IP的美术风格]，[角色实际外貌：发色、服装颜色、标志性装备]，[姿势]，[背景]
```

## 搜索失败时的降级方案

搜不到可用图片时，在 prompt 中把以下信息描述清楚：
- 该 IP 的整体美术风格（写实 3D / 动漫 / 像素等）
- 角色的标志性颜色和轮廓
- 服装材质和关键道具

## 准确度预期

| 条件 | 预期结果 |
|------|---------|
| 有参考图 + prompt 有详细视觉描述 | 风格与关键特征基本吻合 |
| 有参考图 + prompt 描述简略 | 风格匹配，细节可能漂移 |
| 无参考图 + prompt 有详细描述 | 合理近似，无风格保证 |
| 无参考图 + prompt 只写角色名 | 泛化结果，基本不像原版 |

`-r` 是风格迁移，不是精确复刻。参考图越准确、描述越详细，结果越贴近目标。
